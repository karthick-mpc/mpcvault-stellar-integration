Stellar Chain Integration into MPCVault

1. Overview

MPCVault is an institutional-grade MPC custody platform. This integration adds native Stellar support so that MPCVault users — such as Rang — can securely accept and manage payments on the Stellar network, for both XLM and Stellar-issued tokens like USDC.

MPCVault's MPC engine already supports Ed25519 natively, which is the same signing curve Stellar uses. No new cryptographic adapter is needed — Stellar is a natural fit.

Use Case: Accepting Payments on Stellar

Businesses using MPCVault (e.g., Rang) need to receive payments from customers and partners on the Stellar network. This integration enables them to:

  • Create Stellar accounts through MPCVault — secured by MPC, with no single party holding the full private key.
  • Accept XLM and token payments (USDC, yXLM, etc.) with automatic trustline management.
  • Send payments with institutional controls — approval workflows, spending limits, and address allowlists enforced before any transaction is signed.
  • Interact with Soroban dApps for DeFi and advanced contract-based payment flows, all from the same MPC-secured account.
  • Access everything via App, Web, API, or browser plugin — with full transaction history and audit trails.

3. How It Works

Stellar Classic + Soroban

MPCVault supports both layers of the Stellar network. Stellar Classic handles native payments, trustlines, and account management. Soroban enables smart contract interactions for DeFi and advanced payment flows. MPCVault detects the transaction type automatically and routes it to the right network interface.

MPC Key Management

Stellar accounts are generated using MPC-based Ed25519 key generation. The private key is split into three shares — one held by the user and two by MPCVault's cloud infrastructure. The full key is never reconstructed or exposed at any point. The same account works for both Classic and Soroban transactions.

Transaction Lifecycle

Every Stellar transaction follows a consistent lifecycle through MPCVault:

StepWhat Happens
RequestUser initiates a transaction via App, Web, API, Chrome Extension, or WalletConnect.
DecodeTransaction data is decoded into a human-readable format — showing amounts, destinations, assets, fees — so approvers can see exactly what they're signing.
Policy CheckMPCVault's off-chain policy engine validates the transaction against address allowlists, spending limits, risk rules, and multi-party approval requirements. Rejected transactions never reach signing.
SimulateThe transaction is simulated to estimate fees and confirm it will succeed before committing.
ApproveHuman approvers confirm manually, or the API approves programmatically on behalf of authorized users.
MPC SignThe three key shares produce a single valid Ed25519 signature via MPC — without any party ever seeing the full private key.
SubmitThe signed transaction is broadcast to the Stellar network. Confirmation takes ~5 seconds. If submission fails transiently, MPCVault supports controlled re-broadcast.

Policy & Governance

All Stellar transactions are subject to MPCVault's institutional governance controls: address allowlists and denylists, per-transaction and cumulative spending limits, role-based multi-party approval workflows, risk classification, and time-based access controls. These policies are enforced at the MPC signing layer, before any transaction reaches the Stellar network.

Trustline & Asset Management

Stellar requires explicit opt-in (trustlines) to hold non-XLM assets. MPCVault manages trustline creation and removal, governed by asset allowlists and approval policies. Users can hold and transact XLM, USDC, and any other Stellar-issued token.

4. Access Channels

All Stellar functionality is available consistently across every MPCVault access point:

ChannelDescription
MPCVault AppMobile application for transaction management and approvals

MPCVault APIProgrammatic access for developers and enterprise integrations
Chrome ExtensionBrowser plugin for interacting with Stellar dApps
WalletConnectStandard protocol for connecting to external Stellar and Soroban applications

The Chrome Extension and WalletConnect relay dApp requests into MPCVault, where the same policy engine, approval workflows, and MPC signing apply. Developers can integrate Stellar payments into their own systems via the API.

5. Custom Transactions

For advanced users and developers, MPCVault supports custom transaction submission — any protocol-compliant Stellar Classic or Soroban payload. Custom transactions still go through the full governance lifecycle: decode, policy check, simulation, approval, and MPC signing. This enables flexibility without bypassing security controls.

6. Security

Key security properties of the integration:

  • No single point of key compromise — private keys are split across three shares and never reconstructed.
  • Policy enforcement before signing — unauthorized transactions are rejected before the MPC ceremony begins.
  • Cross-chain replay protection — Stellar's network passphrase binding is complemented by MPCVault's chain isolation.
  • Transaction simulation — every transaction is simulated before signing to prevent unexpected outcomes.
  • Asset controls — allowlists and denylists prevent exposure to malicious or unwanted tokens.
  • Full audit trail — unified transaction history across Classic and Soroban with human-readable decoding.
